# Фаза 5 — Изменения в `wgHelper.ts`

## Файл

`src/server/utils/wgHelper.ts`

## Цели

1. `Endpoint` в клиентском WG-конфиге заменить на `127.0.0.1:<clientWgLocalPort>` (обфускатор на клиенте слушает локально).
2. В серверном WG-интерфейсе оставить `ListenPort = 51820`, но добавить firewall-правила в `PostUp`/`PreDown`, ограничивающие приём WG-пакетов только с `127.0.0.1` (внешний приём — через обфускатор).

## `generateClientConfig` — diff

### Было

```ts
return `[Interface]
PrivateKey = ${client.privateKey}
Address = ${address}
MTU = ${client.mtu}
${extraLines.length ? `${extraLines.join('\n')}\n` : ''}
[Peer]
PublicKey = ${wgInterface.publicKey}
PresharedKey = ${client.preSharedKey}

PersistentKeepalive = ${client.persistentKeepalive}

```

### Стало

```ts
return `[Interface]
PrivateKey = ${client.privateKey}
Address = ${address}
MTU = ${client.mtu}
${extraLines.length ? `${extraLines.join('\n')}\n` : ''}
[Peer]
PublicKey = ${wgInterface.publicKey}
PresharedKey = ${client.preSharedKey}
AllowedIPs = ${(client.allowedIps ?? userConfig.defaultAllowedIps).join(', ')}
PersistentKeepalive = ${client.persistentKeepalive}
Endpoint = 127.0.0.1:${wgInterface.clientWgLocalPort}`;
```

### Почему

Клиент после установки запускает локальный `wg-obfuscator`, который слушает на `127.0.0.1:<clientWgLocalPort>` (`source-lport` в клиентском obfuscator-конфиге) и пересылает в `<serverPublicIpV4>:<obfuscatorExtPort>`. WG-клиент должен отправлять пакеты в локальный обфускатор, отсюда `Endpoint = 127.0.0.1:<clientWgLocalPort>`.

### Поля `userConfig.host`/`userConfig.port`

Больше не участвуют в `Endpoint`. Они остаются в БД и UI как справочное значение публичного hostname/port (используется, например, в письме пользователю «ваш сервер — `X:Y`», в Zod-валидации setup-флоу). Удалять не нужно — они не рудимент, а часть identity сервера.

## `generateServerInterface` — diff

### Было

```ts
return `# Note: Do not edit this file directly.
# Your changes will be overwritten!

# Server
[Interface]
PrivateKey = ${wgInterface.privateKey}
Address = ${address}
ListenPort = ${wgInterface.port}
MTU = ${wgInterface.mtu}
${extraLines.length ? `${extraLines.join('\n')}\n` : ''}
PreUp = ${iptablesTemplate(hooks.preUp, wgInterface)}
PostUp = ${iptablesTemplate(hooks.postUp, wgInterface)}
PreDown = ${iptablesTemplate(hooks.preDown, wgInterface)}
PostDown = ${iptablesTemplate(hooks.postDown, wgInterface)}`;
```

### Стало

```ts
const postUpExtra = [
  iptablesTemplate(hooks.postUp, wgInterface),
  `iptables -I INPUT 1 -p udp --dport ${wgInterface.port} ! -s 127.0.0.1 -j DROP`,
].filter(Boolean).join('; ');

const preDownExtra = [
  `iptables -D INPUT -p udp --dport ${wgInterface.port} ! -s 127.0.0.1 -j DROP`,
  iptablesTemplate(hooks.preDown, wgInterface),
].filter(Boolean).join('; ');

return `# Note: Do not edit this file directly.
# Your changes will be overwritten!

# Server
[Interface]
PrivateKey = ${wgInterface.privateKey}
Address = ${address}
ListenPort = ${wgInterface.port}
MTU = ${wgInterface.mtu}
${extraLines.length ? `${extraLines.join('\n')}\n` : ''}
PreUp = ${iptablesTemplate(hooks.preUp, wgInterface)}
PostUp = ${postUpExtra}
PreDown = ${preDownExtra}
PostDown = ${iptablesTemplate(hooks.postDown, wgInterface)}`;
```

### Почему так, а не bind на 127.0.0.1

Альтернатива «`ListenPort = 51820` + bind-only-loopback через `netlink`/`sysctl`» — невозможна (WG не поддерживает bind-адрес, только порт). Единственный способ — INPUT-правило, блокирующее внешний приём на `51820`.

Правило `-I INPUT 1` вставляется в начало цепочки, до любых более ранних `ACCEPT`. `! -s 127.0.0.1` — всё, что пришло не с loopback, дропается.

Для IPv6 аналогично:

```
ip6tables -I INPUT 1 -p udp --dport 51820 ! -s ::1 -j DROP
```

Добавляется в `postUpExtra` парой к IPv4.

### Итоговый `postUpExtra`

```ts
const postUpExtra = [
  iptablesTemplate(hooks.postUp, wgInterface),
  `iptables -I INPUT 1 -p udp --dport ${wgInterface.port} ! -s 127.0.0.1 -j DROP`,
  `ip6tables -I INPUT 1 -p udp --dport ${wgInterface.port} ! -s ::1 -j DROP`,
].filter(Boolean).join('; ');

const preDownExtra = [
  `iptables -D INPUT -p udp --dport ${wgInterface.port} ! -s 127.0.0.1 -j DROP`,
  `ip6tables -D INPUT -p udp --dport ${wgInterface.port} ! -s ::1 -j DROP`,
  iptablesTemplate(hooks.preDown, wgInterface),
].filter(Boolean).join('; ');
```

## Нет изменений в `generateServerPeer`

Серверный peer-блок (секции `[Peer]` в серверном конфиге) не меняется — обфускатор просто туннелирует трафик, не участвует в WG handshake с точки зрения сервера.

## Нет изменений в `wg.up/down/sync/dump`

Управление WG-интерфейсом остаётся через `wg-quick`/`awg-quick` как сейчас.

## Тесты

`src/test/unit/wgHelper.spec.ts` — обновить существующие и добавить:

```ts
describe('generateClientConfig (obfuscated mode)', () => {
  it('uses loopback endpoint with clientWgLocalPort', () => {
    const iface = mockInterface({ clientWgLocalPort: 13255 });
    const client = mockClient();
    const userConfig = mockUserConfig({ host: 'public.example.com', port: 51820 });

    const cfg = wg.generateClientConfig(iface, userConfig, client);

    expect(cfg).toContain('Endpoint = 127.0.0.1:13255');
    expect(cfg).not.toContain('public.example.com');
  });
});

describe('generateServerInterface (loopback-only)', () => {
  it('adds iptables DROP for non-loopback on listen port', () => {
    const iface = mockInterface({ port: 51820 });
    const hooks = mockHooks();

    const cfg = wg.generateServerInterface(iface, hooks);

    expect(cfg).toMatch(/iptables -I INPUT 1 -p udp --dport 51820 ! -s 127\.0\.0\.1 -j DROP/);
    expect(cfg).toMatch(/ip6tables -I INPUT 1 -p udp --dport 51820 ! -s ::1 -j DROP/);
    expect(cfg).toMatch(/PreDown.*iptables -D INPUT -p udp --dport 51820/);
  });
});
```
